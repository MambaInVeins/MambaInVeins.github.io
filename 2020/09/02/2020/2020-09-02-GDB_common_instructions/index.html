IDA/x64dbg/GDB常用快捷键指令

发表于2020-09-02|更新于2021-05-25|debug
|字数总计:746|阅读时长:2分钟|阅读量:

IDA

快速键 说明
左键点两下 跳过去 object / function 所在位址
g 跳到指定位址
x 寻找参考点（那里使用到现在指标指向的object）
esc 返回上一个地方
F5 反编译
n 重新命名

x64dbg

快速键 说明
Ctrl-G 跳到指定位址，可以输入 API Name 或是记忆体位址
Ctrl-F2 重新执行正在被 debug 的程序
F2 在现在选取的拿一行 assembly 下断点
F7 步入 call
F8 步进（执行完 call 指令，不跟进去）
F9 直接执行
Ctrl-F9 执行到 ret 停下来

GDB

PEDA https://github.com/longld/peda

指令 说明 范例
disas addr 对地址addr处的指令进行反汇编，addr可以是函数名字 disas main
b func 在 func 下断点 b main
b *0xADD 在 ADDR 下断点 b *0x08048000
enable 激活断点
disable 禁用断点
info b 查看断点 i b
del $NUM 删除第 $NUM 个断点 d 1
c 继续执行
r 开始执行 / 重新执行
r args 开始执行（给被debug的程序参数）
si step into（跟进去 call）
ni next instruction（不跟进去 call）
x 印出 pointer 指向的资料（help x看详细用法） x/16xw $esp
p 直接印出资料 p/d 0x5566
i b 列出所有 breakpoints (info braekpoints)
i file 列出档案资讯
q 关闭 gdb
set *(char)0xadd = 5 写入记忆体，语法类似 C
set $rax = 0x87 改变 register 的值（也可以改 EIP / RIP）
fini 执行到 ret （离开现在的 function） finish
bt backtrace，印出 function call 的记录 backtrace
set args 可指定运行时参数 set args 10 20 30 40 50
show args 可以查看设置好的运行参数
stack 100 插件提供的，显示栈中100项
find xxx 快速查找，很实用
file app 载入指定的程序.如果刚开始运行gdb而不是用gdb app启动的话可以这样载入app程序
vmmap 查看映射状况 vm

x 命令 查看内存

格式: x /nfu <addr>
说明：
    x 是 examine 的缩写
    n表示要显示的内存单元的个数
    f表示显示方式, 可取如下值
        x 按十六进制格式显示变量。
        d 按十进制格式显示变量。
        u 按十进制格式显示无符号整型。
        o 按八进制格式显示变量。
        t 按二进制格式显示变量。
        a 按十六进制格式显示变量。
        i 指令地址格式
        c 按字符格式显示变量。
        f 按浮点数格式显示变量。
    u表示一个地址单元的长度
        b表示单字节
        h表示双字节
        w表示四字节
        g表示八字节
常用组合：
    x/wx $esp 　　以4字节16进制显示栈中内容
    x/wx 地址　　查看DWORD
    x/c 地址　　单字节查看
    x/16x $esp+12 查看寄存器偏移
    x/x $esp 查看esp寄存器中的值
    x/s addr 查看addr处的字符串
    x/b addr 查看addr处的字符
    x/i addr 查看addr处的反汇编结果
