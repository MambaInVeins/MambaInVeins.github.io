linux强制core dump

发表于2021-06-14|更新于2021-09-15|安全基础知识linux
|字数总计:470|阅读时长:1分钟|阅读量:

打开core文件限制

a.sudo vi /etc/profile

b.文件末尾添加ulimit -c unlimited

source /etc/profile

把文件重新加载到内存

c.root@ubuntu:~/code# ulimit -c

unlimited

说明core文件限制已经去处。

让core文件生成在进程当前目录

sudo su

echo “core-%e-%p-%t” > /proc/sys/kernel/core_pattern

设置永久保存

上面的设置都只是临时的，重启之后就需要重新设置，如何设置永久生效呢？

打开/etc/security/limits.conf 文件，在该文件的最后加上两行

#下面是我的配置

@root soft core unlimited

@root hard core unlimited

命名规则的修改在/proc/sys/kernel/core_pattern中也只是临时的，这个也是动态加载和生成的。永久修改在/etc/sysctl.conf文件中，在该文件的最后加上两行：

kernel.core_pattern = /var/core_log/core_%e_%t_%p

kernel.core_uses_pid = 0

可以使用以下命令，使修改结果马上生效。

#sysctl –p

问题归总

1.上述操作都做了还是不行？

重启试试

2.coredump文件大小为0？

经常会发现生成的core文件大小为0，原因也各种各样，首先要用ulimit -c命令查看是否大小被限制，其次要确定指定路径大小是否够用，比如我们指定的/home目录；再次就是在共享文件夹中生成的coredump为0，具体原因无法知晓，放到宿主机目录中就可以。

gdb调试coredump

gdb程序

gdb ./test

进入gdb环境后，敲

core-file core.test.xxxx

查看堆栈使用bt或者where命令

bt
where

没有调试信息的情况下找core的代码行,frame addr(帧数)或者简写如上，f 0 跳转到core堆栈的第0帧。

disassemble 打开该帧函数的反汇编代码。

之后info frame查看堆栈寄存器信息

info frame
